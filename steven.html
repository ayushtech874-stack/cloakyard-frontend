Cloakyard - Stock Manager Guide

This guide explains how to properly sync WooCommerce data with the custom Next.js storefront.

1. How to Set Up Pricing (Normal vs Sale Price)

For Simple Products

If a product does not have sizes or variations:

  • Go to Product Data > General in WooCommerce.
  • Enter your Regular Price (e.g., 1499).
  • Enter your Sale Price (e.g., 899).

The website will automatically cross out 1499 and display 899.

For Variable Products (Products with Sizes like S, M, L)

Variable products require prices to be set on the variations themselves.

  • Go to Product Data > Variations.
  • Expand a variation (e.g., Size M).
  • You MUST enter both the Regular Price AND the Sale Price directly inside the variation settings.
  • If you only enter the Regular Price (e.g. 899) and leave Sale Price blank, the website will think 899 is the normal price and will not cross anything out.
  • Pro Tip: You can bulk edit all variations at once by clicking the dropdown in the Variations tab and selecting "Set regular prices", then "Set sale prices".

2. How to Manage Filters (Like adding "Color")

Because Cloakyard uses a highly customized React storefront, the sidebar filters (Fit, Size, Gender) are hardcoded to match the exact design aesthetic. If you want to add a completely new filter (like "Color"), it requires a minor code update.

Step A: Setup in WooCommerce

  • Go to Products > Attributes in WooCommerce.
  • Add a new Attribute called Color.
  • Go to a product and assign the Color attribute (e.g., Black, White).

Step B: Update Frontend Code

To make the filter show up on the website's sidebar, edit app/(public)/shop/page.tsx:

<div className="space-y-4">
  <h3 className="font-headline-lg text-xl text-on-background uppercase tracking-wider">COLOR</h3>
  <div className="flex flex-col gap-2">
    {['black', 'white', 'grey'].map(c => (
      <label key={c} className="flex items-center gap-3 cursor-pointer group">
        <input type="checkbox" onChange={() => toggleArrayFilter('color', c, colors)} />
        {c}
      </label>
    ))}
  </div>
</div>

Step C: Update Backend API

To make the API actually filter the products, edit app/api/products/route.ts:

const filterColor = searchParams.get('color')?.toLowerCase().split(',')
if (filterColor && filterColor.length > 0) {
  formatted = formatted.filter((p: any) => filterColor.includes(p.colour.toLowerCase()))
}
